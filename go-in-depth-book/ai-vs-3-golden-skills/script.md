# AI  vs 21世纪三大黄金技能

大家好

我最近在看这本书《图解大模型 生成式AI的原理与实践》。这是一本翻译的书，译者在序里面说这本的翻译是在大模型的辅助下完成的。这件事情对我的冲击很大。

有的人会问，别人用大模型翻译一本书，怎么就冲击到你一个程序员了呢？
熟悉我的朋友可能知道，我的专业是英语。哥们也是211重点院校英语本科毕业，手握英语8级证书的人。只不过毕业的时候，我的同学们都选择了考研、考博，雅、思出国，而我选择成为一个名光荣的程序员，加入咱们码农大家庭。

我为什么会有这样的选择，这就不得不从上世纪8、90年代说起。我们那时候社会上流行一种说法，未来21世纪3大黄金技能：英语、电脑、和开车。虽然说现在这三项已经不值得一提，但当年在各大媒体上“21世纪3大黄金技能”的热度和现在“AI要取代人类”的热度可不相上下。

我当时的想法是，电脑是我的爱好，我可以自学，但英语不行，我需要有人教我。于是我就选择了英语专业。上学期间，我除了完成正常的学习，几乎所有的课余时间都用在自学编程上了。于是毕业时就顺利找到了工作，成为一名码农。

因此，在我心目中，英语一直是我的一个“兜底”技能。我不止一次想到过，如果写不了代码了，大不了我去搞翻译嘛。这可好。现在的大模型AI直接把我这个兜底的技能给干碎了。

说到AI，其实之前我一致是很排斥的。因为它的幻觉，因为他的不稳定性。给大家说个内幕消息，我的这本书在校对期间，我一度怀疑出版社使用AI来修改稿件，我很生气，还差点因为这个和出版社闹翻。但凡当时我再冲动那么一点点，或者我情商在低那么一丢丢，这本书就夭折了。

当然，后来随着我越来越多地使用AI，越来越多地了解AI，我也和AI和解了。这不，我现在都开始研究AI的原理了。为什么要研究AI的原理，这和我的性格有关，凡是我关心的事情，我都要弄个水落石出。退一万步说，就算AI要取代我，浪潮要把我拍死在沙滩上，我也要死个明白，我也要知道这个对手AI你是谁，你倒是什么，到底什么原理。

回到这个“21世纪三大黄金技能”上。开车自然不用说，现在严格意义上已经不算是一门技能了；英语，特别是翻译，现在AI在书面翻译方面已经很厉害了，这本专业书籍的翻译就是一个例子，据说在同声传译领域也有非常惊人的进展。最后说说电脑，咱们最关心的编程领域，无论是vibe coding还是serious coding，AI的表现都是我们有目共睹的。

现在有很多人在讨论“AI会不会取代人类，会不会取代人类的工作”。我现在的想法是：我们的工作和生活离不开AI，但也同样离不开人类。现在AI的幻觉，AI的不稳定性和不可解释性都是阻碍AI全面接管我们工作和生活的绊脚石。在很长一段时间内，特别是在编程领域，AI和程序员是要和谐共存的，AI是需要人类来把关和兜底的。

但是，AI的接入，依然不可避免地会淘汰掉一部分人类，淘汰掉一部分低端劳动力。

任何一个领域，最顶尖的那20%的人是永远不会被淘汰的。

就以这本书的翻译为例，既然AI翻译这么厉害，那么这本书引进中文版时，为什么找到了这个译者，而不是你，也不是我？这还不是因为这位译者对AI原理的深刻理解，和对英语语言的熟练使用吗？像他这样的高端人才会担心自己被淘汰吗？

我的父亲是一名电气工程师，他曾经教导我说“你不可能让所有人都喜欢你，但你可以让你的工作岗位离不开你”。什么意思？任何一个领域，任何一个项目都需要技术专家，救火队员。当这个项目有重大决策时，会想到你，当项目遇到重大问题时，还是会想到你。那么你就是不可替代的。你就是大动脉，如果把你裁了，这个项目就麻烦了。

相反，如果我们只是一个AI的熟练使用者，那么再培养一个同样可以熟练使用AI的人，我想并不是什么难事吧？所以，打铁还得自身硬，自己掌握的核心原理和专业技能才是我们安身立命的根本。

另外，像我们这样的程序员，把自己最美好的青春岁月奉献给了世界编程事业，结果到了上有老下有小的年纪，让咱们转行，干什么去？送外卖？扫大街？不现实啊，我们只有在自己熟悉的领域继续深耕，向着技术专家的方法发展。

那么在成为技术专家的路上，AI要扮演什么角色，我认为不能是拐杖而应该是老师。虽然AI偶尔会胡说八道，但它也总能在不经意间给我们灵感，写出我们从来没想到过的代码，输出我们从来没接触过的知识。我们要做的就是甄别哪些是值得学习的，把它的原理搞清楚，把这些知识内化为我们自己的技能。

现在在围棋领域，棋手们都开始通过和AI对弈来学习之前从来没有见过的招式，我想我们程序员也是一样的道理。
